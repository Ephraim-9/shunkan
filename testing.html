Test Plan & Test Cases

A comprehensive test strategy covering unit tests, integration tests, platform-specific clipboard adapter tests, and Android instrumented tests with verification commands and expected outputs.

⚡ Quick Test Commands

Run these commands from the project root (/shunkan) to execute the full test suite:

# Run all workspace tests
cargo test --workspace

# Run only core library tests
cargo test -p shunkan-core

# Run with output (for debugging)
cargo test -p shunkan-core -- --nocapture

# Run a specific test
cargo test -p shunkan-core --test test_mdns_peer_discovery -- --nocapture

# Run clippy lint checks
cargo clippy --workspace -- -D warnings

# Run tests with coverage (requires cargo-llvm-cov)
cargo llvm-cov --workspace --html

🧪 Unit Tests — Core Modules (shunkan-core)

Each core module has dedicated unit tests validating its public API and internal logic. Tests are located in core/tests/ and inline #[cfg(test)] modules.

Discovery Module (discovery.rs)

TC-101: mDNS Service Registration Pending

Verify that calling register_service() correctly advertises _shunkan-sync._udp.local. with hostname, OS type, and port 4433 in TXT records.

Command cargo test -p shunkan-core test_mdns_service_registration -- --nocapture

Expected: Service registered successfully. TXT record contains os=Linux and port=4433.

TC-102: mDNS Peer Discovery Pending

Start two ServiceDaemon instances on loopback. Verify that each discovers the other's service record within 2 seconds and emits PeerDiscovered events.

Command cargo test -p shunkan-core --test test_mdns_peer_discovery -- --nocapture

Expected: Both peers discovered. Event payloads contain correct hostname and IP address.

Transport Module (transport.rs)

TC-201: QUIC Server Startup Pending

Verify that a QUIC server endpoint binds to 0.0.0.0:4433 with self-signed TLS cert and accepts incoming connections.

Command cargo test -p shunkan-core test_quic_server_startup -- --nocapture

Expected: Server bound. Logs show QUIC endpoint listening on 0.0.0.0:4433.

TC-202: QUIC Encrypted Handshake Pending

Spin up a server and client on loopback. Client connects with TLS 1.3, verify successful handshake and bidirectional stream opening.

Command cargo test -p shunkan-core --test test_quic_encrypted_handshake -- --nocapture

Expected: Handshake completes in <100ms. Both sides can open streams. TLS version: 1.3.

TC-203: Multiplexed Stream Independence Pending

Open two concurrent streams — send a large payload on Stream 0 and a small payload on Stream 1. Verify Stream 1 completes before Stream 0 (no head-of-line blocking).

Command cargo test -p shunkan-core test_stream_multiplexing -- --nocapture

Expected: Stream 1 (clipboard) completes in <10ms while Stream 0 (file) is still transferring.

Protocol Module (protocol.rs)

TC-301: Frame Serialization Round-Trip Pending

Serialize each message type (ClipboardItem, FileChunkHeader, HandshakeRequest) to bytes via bincode, then deserialize back. Assert field equality.

Command cargo test -p shunkan-core test_frame_roundtrip -- --nocapture

Expected: All frame types serialize and deserialize identically. No data loss.

Crypto Module (crypto.rs)

TC-401: X25519 Key Exchange Pending

Generate two ephemeral key pairs. Perform Diffie-Hellman exchange. Verify both parties derive the same shared secret.

Command cargo test -p shunkan-core test_x25519_key_exchange -- --nocapture

Expected: shared_secret_a == shared_secret_b. 32 bytes length.

TC-402: PIN Code Generation & Verification Pending

Generate a 6-digit PIN from the shared secret. Verify it matches on both sides and is always 6 digits (zero-padded).

Command cargo test -p shunkan-core test_pin_verification -- --nocapture

Expected: PIN is 6 digits. Both peers compute identical PIN from shared secret.

History Module (history.rs)

TC-501: SQLite Insert & Retrieve Pending

Insert 10 clipboard items. Retrieve all and verify ordering (newest first). Verify BLAKE3 hash is computed and stored.

Command cargo test -p shunkan-core test_history_insert_retrieve -- --nocapture

Expected: 10 items stored. Retrieved in reverse-chronological order. Each has a 32-byte BLAKE3 hash.

TC-502: LRU Eviction Pending

Set LRU limit to 5. Insert 8 items. Verify only the 5 most recent remain and the oldest 3 are evicted.

Command cargo test -p shunkan-core test_history_lru_eviction -- --nocapture

Expected: Database contains exactly 5 entries. Oldest 3 entries deleted.

TC-503: BLAKE3 Deduplication Pending

Insert the same text twice. Verify only one entry exists (deduplicated by BLAKE3 hash). Timestamp should be updated.

Command cargo test -p shunkan-core test_history_deduplication -- --nocapture

Expected: Only 1 entry in database. Timestamp reflects the second insertion time.

🔗 Integration Tests — Discovery + Transport

End-to-end tests that verify the full flow: mDNS discovery → QUIC connection → clipboard sync → file transfer.

IT-001: Full Discovery-to-Clipboard Sync Pending

Start two Shunkan instances on loopback. Instance A discovers Instance B via mDNS. A sends a ClipboardItem to B over QUIC. B receives and stores it in history.

Command cargo test -p shunkan-core --test integration_clipboard_sync -- --nocapture

Expected: Clipboard text "Hello from Device A" appears in Device B's history within 50ms.

IT-002: File Transfer with BLAKE3 Verification Pending

Transfer a 10 MB test file between two loopback instances. Verify BLAKE3 chunk hashes match on both ends. Verify file content integrity byte-for-byte.

Command cargo test -p shunkan-core --test test_large_file_quic_stream -- --nocapture

Expected: File transferred successfully. All chunk BLAKE3 hashes match. Final file SHA matches source.

IT-003: Concurrent Clipboard + File Transfer Pending

Start a large file transfer on Stream 0. While it's in progress, send a clipboard item on Stream 1. Verify the clipboard item arrives immediately without waiting for the file transfer to complete.

Command cargo test -p shunkan-core --test test_concurrent_streams -- --nocapture

Expected: Clipboard arrives in <50ms. File transfer continues uninterrupted.

🐧 Desktop Clipboard Adapter Tests

DT-001: Session Type Detection Pending

Verify detect_session_type() correctly reads $XDG_SESSION_TYPE and returns X11, Wayland, or Unknown.

Command cargo test -p desktop-linux test_session_detection -- --nocapture
DT-002: Clipboard Read/Write (X11) Pending

On an X11 session, write text to clipboard via arboard, read it back, verify content matches. Requires running X server.

Command cargo test -p desktop-linux test_x11_clipboard_readwrite -- --nocapture
DT-003: Feedback Loop Prevention Pending

Write a remote clipboard item with a hash nonce. Verify the listener skips re-broadcasting it (nonce matches last received payload).

Command cargo test -p desktop-linux test_feedback_loop_prevention -- --nocapture
