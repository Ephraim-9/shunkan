Usage, Deployment & Installation Guide

Complete instructions on how to build, install, configure, troubleshoot, and package Project Shunkan across Linux desktop environments and Android mobile devices.

🛠️ Prerequisites & Build Environment

To compile Project Shunkan from source, ensure your development environment satisfies these minimum toolchain requirements:

Linux Desktop Toolchain

  • Rust Toolchain: 1.78+ via rustup (Edition 2021)
  • System Libraries (Debian/Ubuntu):
    sudo apt install build-essential pkg-config libx11-dev libxcb1-dev libxkbcommon-dev libwayland-dev libssl-dev cmake
  • System Libraries (Fedora/RHEL):
    sudo dnf install gcc gcc-c++ pkgconfig libX11-devel libxcb-devel libxkbcommon-devel wayland-devel openssl-devel cmake
  • System Libraries (Arch Linux):
    sudo pacman -S base-devel pkgconf libx11 libxcb libxkbcommon wayland openssl cmake

Android Toolchain & FFI

  • JDK: OpenJDK 17 or higher
  • Android Studio & SDK: API 34+ (Android 14) platform and Build Tools
  • Android NDK: NDK r26d+ for compiling native Rust libraries targeting arm64-v8a and x86_64
  • Rust Android Targets:
    rustup target add aarch64-linux-android x86_64-linux-android
  • cargo-ndk:
    cargo install cargo-ndk

📦 Building from Source

1. Building the Linux Desktop Daemon & UI

From the project root directory, verify and compile all workspace members in release mode:

# 1. Run full test suite across core and desktop
cargo test --workspace

# 2. Build optimized release binaries
cargo build --release --workspace

# The compiled headless daemon binary will be located at:
# ./target/release/shunkan-desktop

2. Building the Android Native Library via UniFFI

Compile the shared Rust core library into native Android shared objects (`libshunkan_core.so`) for mobile devices and emulators:

# Build for physical arm64 Android devices
cargo ndk -t arm64-v8a -o mobile-android/app/src/main/jniLibs build --release -p shunkan-core

# Build for x86_64 Android Studio emulators
cargo ndk -t x86_64 -o mobile-android/app/src/main/jniLibs build --release -p shunkan-core

# Assemble the Android APK using Gradle
cd mobile-android && ./gradlew assembleDebug

🚀 Running the Application

Linux Desktop Daemon

To run the daemon directly in your terminal with debug logging enabled:

RUST_LOG=debug ./target/release/shunkan-desktop

Upon startup, the daemon will automatically:

  • Detect your display session (X11, Wayland wlroots, or GNOME Wayland).
  • Bind to UDP port 4433 for QUIC transport connections.
  • Advertise service _shunkan-sync._udp.local. via mDNS.
  • Monitor clipboard copy events and broadcast BLAKE3 hashes to paired peers.

Opening the Glassmorphic Command Palette

To inspect the standalone webview UI during development without launching a full Tauri container, open the HTML bundle directly in your browser:

xdg-open desktop-linux/src/index.html

⚙️ Configuration & Firewall Rules

Because Project Shunkan operates entirely on the local network without cloud relays (per INV-02), local system firewalls must allow multicast DNS and QUIC traffic.

Required Network Ports:
• 5353 / UDP (mDNS Service Discovery)
• 4433 / UDP (QUIC Encrypted Data Streams)

Configuring UFW (Ubuntu/Debian)

sudo ufw allow 5353/udp comment 'Shunkan mDNS Discovery'
sudo ufw allow 4433/udp comment 'Shunkan QUIC Transport'
sudo ufw reload

Configuring Firewalld (Fedora/RHEL)

sudo firewall-cmd --permanent --add-port=5353/udp
sudo firewall-cmd --permanent --add-port=4433/udp
sudo firewall-cmd --reload

🏷️ Packaging for Production & Distribution

Project Shunkan is packaged into standard platform formats for seamless deployment across distributions:

Debian Package (.deb)

Using `cargo-deb`:

cargo install cargo-deb
cargo deb -p shunkan-desktop
# Output: target/debian/shunkan-desktop_0.1.0_amd64.deb
sudo dpkg -i target/debian/shunkan-desktop_0.1.0_amd64.deb

AppImage Portable Bundle

Using `cargo-appimage`:

cargo install cargo-appimage
cargo appimage -p shunkan-desktop
# Output: target/appimage/shunkan-desktop-x86_64.AppImage
chmod +x target/appimage/*.AppImage

Android APK Package

Using Gradle Wrapper:

cd mobile-android
# Build signed/unsigned release bundle
./gradlew assembleRelease
# Output: app/build/outputs/apk/release/app-release-unsigned.apk

❓ Troubleshooting & FAQ

Q: Peers are not discovering each other across my Wi-Fi network

Diagnosis: Your router or AP may block UDP Multicast (mDNS), or your system firewall has dropped port 5353/udp.

Solution: Verify multicast is enabled by running avahi-browse -rt _shunkan-sync._udp on Linux. Ensure your firewall rules above are applied. If client isolation is enabled on guest Wi-Fi networks, connect both devices to the same trusted LAN subnet.

Q: Clipboard history copies in an infinite loop on X11

Diagnosis: A misbehaved X11 clipboard manager is rewriting the clipboard ownership without modifying the content.

Solution: Shunkan implements BLAKE3 hash deduplication (INV-05) to break feedback loops. Ensure you are running version 0.1.0+ where `last_written_hash` and `last_hash` guards are active in `clipboard.rs`.

Q: TLS Certificate validation fails on initial connection

Diagnosis: Because Shunkan generates ephemeral self-signed certificates via `rcgen`, standard root CA checks must be bypassed in favor of PIN hash pinning.

Solution: Ensure your QUIC client configuration uses the custom `rustls::ClientConfig` returned by `crypto::generate_quinn_configs()`, which includes the self-signed certificate in its explicit root store.
