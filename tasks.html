Phase 1: Shared P2P Core & FFI Bridge (`shunkan-core`)
Foundation
To Do
Initialize the core/ Rust library with Cargo workspace configuration. Setup uniffi dependency and create all six modules with stub exports: discovery.rs, transport.rs, protocol.rs, crypto.rs (X25519 key exchange scaffold), and history.rs (rusqlite + LRU scaffold). Export a stub hello_shunkan() function to verify Kotlin binding generation via UniFFI.
Verification Command cd core && cargo build && cargo test && cargo run --bin uniffi-bindgen generate --library target/debug/libshunkan_core.so --language kotlin --out-dir ../mobile-android/app/src/main/java
To Do
Integrate `mdns-sd` and `local-ip-address` to broadcast service record `_shunkan-sync._udp.local` containing hostname, OS type (`Linux` / `Android`), and active QUIC port. Build background listener that emits discovered/lost peer events.
Verification Command cargo test -p shunkan-core --test test_mdns_peer_discovery -- --nocapture
To Do
Setup `quinn` server and client endpoints with `rustls` self-signed TLS certificates. Implement PIN-verified Diffie-Hellman (`x25519-dalek`) key exchange that stores mutual peer public keys in local persistent storage.
Verification Command cargo test -p shunkan-core --test test_quic_encrypted_handshake -- --nocapture
Phase 2: Linux Desktop & Wayland/X11 Tray (`desktop-linux`)
Desktop UI
To Do
Create Tauri v2 project under `desktop-linux/`. Configure `tauri-plugin-system-tray` to hide main window on boot, display status menu in Linux system tray, and start embedded `shunkan-core` background thread.
Verification Command cd desktop-linux && cargo check && npm run tauri build -- --debug
To Do
Build clipboard adapter using session detection ($XDG_SESSION_TYPE):
X11: Use arboard with a background thread polling at ~100ms intervals for CLIPBOARD and PRIMARY atom changes.
Wayland (wlroots: Hyprland/Sway/KDE): Use wl-clipboard-rs implementing ext-data-control-v1 (preferred) / wlr-data-control-unstable-v1 for privileged background clipboard monitoring.
Wayland (GNOME/Mutter): Run a wl-paste --watch-equivalent loop via wl-clipboard-rs; use zbus + xdg-desktop-portal-gnome backend for authorization prompt on first launch.
When local clipboard changes, push ClipboardItem protobuf frame to shunkan-core broadcasting queue.
Verification Command cargo test -p desktop-linux --test test_linux_clipboard_capture -- --nocapture
To Do
Register global shortcut Super + V (tauri-plugin-global-shortcut) to display searchable history UI. When item is clicked/selected, inject paste using compositor-specific approach:
X11: Use enigo (Xtest extension) to synthesize Ctrl+V.
Wayland (wlroots: Hyprland/Sway/KDE): Use wtype implementing virtual-keyboard-v1 protocol.
Wayland (GNOME/Mutter): Write to clipboard and display a toast notification prompting the user to paste with Ctrl+V (GNOME blocks virtual-keyboard-v1 for security).
Verification Command npm run dev (Manual verify: press Super+V, select item, ensure text pastes into terminal on X11/wlroots; verify clipboard-write + toast on GNOME Wayland)

Mobile UI

To Do

Shizuku Power-User Toggle: Also implement an optional Shizuku privilege elevation path: detect if Shizuku is running and, if authorized, request appops set com.shunkan.sync READ_CLIPBOARD allow to enable standard background OnPrimaryClipChangedListener without requiring the user to switch keyboard.

Note: Do NOT use AccessibilityService for clipboard capture — violates Play Store policies and is silently blocked by Android 10+.
Verification Command ./gradlew connectedAndroidTest
Phase 4: High-Speed File Streaming & Polish
Performance
To Do
Add file transfer protocol frames inside shunkan-core. Stream binary files in 64 KB chunks using BLAKE3 (blake3 crate) for rolling integrity verification — BLAKE3 achieves 15–20 GB/s throughput via SIMD (vs SHA-256 which bottlenecks CPU at Wi-Fi 6 speeds >100 MB/s). Implement Merkle-tree chunked verification for resumable integrity checks. Implement transfer progress event broadcasting and auto-resume from last byte offset on reconnect.
Verification Command cargo test -p shunkan-core --test test_large_file_quic_stream -- --nocapture
