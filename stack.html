Technology Stack & System Architecture

A deep-dive into the hybrid architecture combining a high-performance Rust networking engine (`shunkan-core`) with native OS frontends (`Tauri v2` on Linux and `Jetpack Compose` on Android).

🏆 Recommended Architecture & Comparative Justification

When architecting a tool that must run silently in the background on both Linux desktops and Android phones while maintaining near-zero latency and high memory efficiency, architectural choice determines success or failure:

⭐ Selected Stack

Rust Core + Native UI (Tauri / Kotlin)

Shared Engine: Pure Rust (`shunkan-core`) compiled to native library (`libshunkan_core.so`) for Android via `UniFFI`, and directly linked into Tauri v2 on Linux.

Why Selected: Zero-copy async streams (< 15MB RAM footprint on Linux), sub-millisecond local P2P encryption via `quinn`, and 100% native look & feel on both Android (`Jetpack Compose`) and Linux (`Tauri/GTK4`).

Alternative 2

Flutter + Rust (`flutter_rust_bridge`)

Why Reconsidered: While Flutter allows a single UI codebase across Linux and Android, Flutter Desktop on Linux consumes ~70-100MB of RAM idle, and custom Android Quick Settings / Share Sheet targets still require writing substantial native Kotlin boilerplate regardless.

Alternative 3

Pure Go (LocalSend Style)

Why Reconsidered: Go (`gomobile`) works well for networking, but its garbage collector introduces non-deterministic memory spikes on Linux desktop tray daemons, and CGO/Wayland clipboard bindings on Linux are significantly more mature in the Rust ecosystem (`wl-clipboard-rs`, `zbus`, `arboard`).

🦀 Shared Core Engine (`shunkan-core`)

The foundational Rust crate shared by both operating systems. It decouples networking, cryptography, and protocol handling entirely from UI frameworks.

⚡
Transport & Multiplexing

Quinn (QUIC Protocol over UDP)

Provides 0-RTT/1-RTT connection setup and concurrent streams so lightweight clipboard text items never get blocked behind a 5GB file transfer.

🔍
Local Discovery

mdns-sd & local-ip-address

Zero-configuration multicast DNS advertising and discovering `_shunkan-sync._udp.local` service records across local Wi-Fi / subnet interfaces.

🔐
Cryptography & Handshake

rustls + x25519-dalek + ring

Modern TLS 1.3 encryption using Diffie-Hellman ephemeral key exchange verified via 6-digit PIN / QR pairing.

📦
Serialization & FFI

Serde + UniFFI

High-speed binary and JSON serialization (`serde_json` / `bincode`) auto-generating clean, type-safe Kotlin interface bindings via Mozilla's `UniFFI`.

🐧 Linux Desktop Architecture (`desktop-linux`)

Built to run silently in the system tray and replace the native clipboard manager on both X11 and Wayland compositors.

🖼️
Desktop Framework

Tauri v2 (Rust + Webview / Native Tray)

Provides an ultra-lightweight system tray icon (`tauri-plugin-system-tray`) and pops up a glassmorphic HTML/CSS/JS command palette on `Super + V` hotkey.

📋
Clipboard OS Bindings

arboard + wl-clipboard-rs + zbus

Auto-detects `$XDG_SESSION_TYPE`. Uses `arboard` on X11, `wl-clipboard-rs` for `wlr-data-control`, and `zbus` for XDG Desktop Portal clipboard IPC on strict Wayland setups.

⌨️
Global Hotkeys & Automation

tauri-plugin-global-shortcut + enigo

Listens for custom key combinations to trigger the clipboard history popup and synthesizes keypresses (`Ctrl+V`) when a user selects a history item to paste.

📱 Android Mobile Architecture (`mobile-android`)

Specifically crafted to comply with Android 10+ (API 29+) background restrictions without requiring root permissions, while offering optional power-user IME features.

Modern Native UI

Kotlin + Jetpack Compose + Material 3: Fluid, reactive Android frontend that embeds `libshunkan_core.so` via `UniFFI`. Displays connection status, transfer speeds, and local clipboard history card grids.

OS System Hooks

AndroidX ShareTarget & TileService: Registers as a direct system share option (`Send to Linux Desktop`). Provides a Quick Settings tile in the notification shade to pull/push clipboard history in one tap.

Optional Background IME

Custom InputMethodService: For power users desiring true background auto-capture on Android 10+, provides an optional lightweight keyboard overlay with direct `ClipboardManager` listener access.

📂 Project Codebase Layout & Directory Structure

Future AI agents should maintain this clean, modular workspace hierarchy across all development stages: 

/shunkan
├── core/                        # Shared Rust P2P & Clipboard Engine (shunkan-core)
│   ├── Cargo.toml
│   ├── src/
│   │   ├── lib.rs               # Core FFI exports via UniFFI
│   │   ├── discovery.rs         # mDNS-SD local network broadcaster & listener
│   │   ├── transport.rs         # QUIC / TLS 1.3 socket manager & stream multiplexer
│   │   ├── protocol.rs          # Message frames (ClipboardItem, FileChunk, Handshake)
│   │   ├── crypto.rs            # X25519 key exchange, PIN verification, cert storage
│   │   └── history.rs           # SQLite / In-memory LRU clipboard history store
│   └── uniffi/                  # UniFFI interface definition (.udl or macro-based)
├── desktop-linux/               # Linux Desktop Application (Tauri v2 + Rust)
│   ├── src-tauri/
│   │   ├── Cargo.toml           # Links to ../core
│   │   ├── src/
│   │   │   ├── main.rs          # System tray bootstrap & hotkey registration
│   │   │   ├── clipboard.rs     # X11/Wayland clipboard listener & auto-injector
│   │   │   └── commands.rs      # Tauri IPC commands bridged to shunkan-core
│   │   └── tauri.conf.json
│   └── src/                     # HTML/CSS/JS Glassmorphic Command Palette UI
│       ├── index.html
│       ├── styles.css
│       └── app.js
└── mobile-android/              # Android Native Application (Kotlin + Jetpack Compose)
    ├── build.gradle.kts
    └── app/src/main/
        ├── jniLibs/             # Compiled libshunkan_core.so (arm64-v8a, x86_64)
        ├── java/com/shunkan/sync/
        │   ├── MainActivity.kt  # Jetpack Compose UI (History, Pairing, Settings)
        │   ├── ShareActivity.kt # Receives text/files from Android Share Sheet
        │   ├── ShunkanTile.kt     # Quick Settings TileService for 1-tap sync
        │   ├── ShunkanIME.kt      # Optional custom keyboard for true background sync
        │   └── core/            # Auto-generated UniFFI Kotlin wrapper classes
        └── res/                 # Vector drawables, themes, XML configs
